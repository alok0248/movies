The endpoint only returns the configured JSON when Basic Auth, app identity, and build ID match this saved Android app.
1. Request
Method
GET
Endpoint
{{ app_endpoint }}
App identity header
X-Android-App: [your-app-identity]
Supports comma-separated lists (e.g., #225, #226, #227) and ranges (e.g., #225-#250)
Build header
X-Android-Build: [your-build-id]
Supports comma-separated lists and ranges (e.g., 1-10)
User ID header
X-Android-User-ID: [your-android-id-or-uuid]
Unique device identifier for tracking unique visitors (optional but recommended)
Device model header
X-Android-Device: [device-model]
Device model name (e.g., Pixel 7 Pro, Nexus 5)
OS version header
X-Android-OS-Version: [os-version]
Android OS version (e.g., 13, 14)
All parameters can also be sent as query params: ?app=[app-id]&build=[build-id]&user_id=[user-id]&device=[device-model]&os=[os-version]
2. Authentication

Send HTTP Basic Authentication with the saved endpoint credentials.

Authorization: Basic {{ basic_auth_value }}
3. Full cURL Example
curl -u {{ android_app.access_username }}:{{ android_app.access_password }} \
  -H "X-Android-App: {{ android_app.allowed_endpoint }}" \
  -H "X-Android-Build: {{ android_app.allowed_build_id|default:'your-build-id' }}" \
  "{{ app_endpoint }}"
200: Configuration response

Returned when endpoint credentials, app identity, and build are accepted.

{{ sample_success_payload }}
426: Update required

Returned when the supplied build ID does not match the saved build ID.

{{ sample_update_payload_json }}
4. Android handling
On HTTP 200, parse the JSON payload and continue normally. On HTTP 426, parse the update payload, display its message, and open the returned apk_url when available. HTTP 403 means the credentials or identity were rejected. HTTP 404 means the endpoint is disabled or does not exist.
